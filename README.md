Ramas (Branches) en Git

Las ramas en Git son líneas de trabajo independientes dentro de un mismo proyecto.
Permiten que varias personas trabajen al mismo tiempo en diferentes funcionalidades o cambios sin afectar el código principal.

¿Para qué sirven las ramas?

Las ramas sirven para:

Desarrollar nuevas funciones sin dañar el proyecto principal.
Probar cambios de manera segura.
Corregir errores sin afectar otras partes del sistema.
Trabajar en equipo de forma organizada.
Mantener versiones estables del proyecto.

La rama principal normalmente se llama main o master, y contiene la versión más estable del proyecto.

¿Cómo se usarán las ramas en nuestro proyecto?

En el proyecto utilizaremos ramas para organizar el trabajo de cada integrante y evitar conflictos en el código.

Ejemplo de uso:
main
Contendrá la versión estable y funcional del sistema.
rama-login
Se utilizará para desarrollar el módulo de inicio de sesión.
rama-base-datos
Será usada para trabajar todo lo relacionado con la base de datos.
rama-interfaz
Se enfocará en el diseño y apariencia visual del sistema.

Cada integrante trabajará en su propia rama y, cuando termine sus cambios y todo funcione correctamente, esos cambios se unirán a la rama principal mediante un proceso llamado merge.

Beneficios de usar ramas en el proyecto
Mayor organización del trabajo.
Menor riesgo de dañar el proyecto principal.
Facilidad para identificar errores.
Mejor colaboración entre los integrantes del equipo.
Posibilidad de trabajar varias funciones al mismo tiempo.
Conclusión

El uso de ramas en Git es fundamental para desarrollar proyectos de manera profesional y organizada. Gracias a ellas, el equipo puede trabajar simultáneamente en diferentes partes del sistema sin generar conflictos, manteniendo siempre una versión estable y segura del proyecto.
